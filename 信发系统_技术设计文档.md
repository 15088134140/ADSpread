## 1. 系统概述

### 1.1 项目背景

信发系统是一套面向连锁门店的广告信息发布系统，由Web管理后台和安卓客户端两大部分组成，实现：
- 集中管理多门店、多设备的广告内容
- 支持多种分屏布局展示不同广告内容
- 定时生效和远程更新广告节目
- 安卓客户端7x24小时稳定运行
- v1.3新增：门店行业分类管理、素材审核流程

### 1.2 业务目标

| 目标 | 说明 |
|------|------|
| 统一管理 | 集中管理1000+门店，5000+设备 |
| 分屏展示 | 支持横屏/竖屏多种分屏布局 |
| 实时推送 | 广告更新秒级生效 |
| 稳定运行 | 客户端长时间稳定运行，开机自启动 |
| 多语言支持 | 日语、中文简体、英语三种语言 |
| 行业分类 | 支持8种行业分类管理门店 |
| 素材审核 | 完善的素材审核机制，确保内容合规 |

## 2. 整体架构设计

### 2.1 架构分层

```
┌─────────────────────────────────────────────────────────┐
│                     客户端层                              │
├─────────────┬───────────────────┬───────────────────────┤
│ 管理后台Web │  安卓展示客户端   │   设备设置APP         │
│ Vue3 + TS   │  Kotlin原生       │                       │
└─────────────┴───────────────────┴───────────────────────┘
                              ↓ HTTPS / WebSocket
┌─────────────────────────────────────────────────────────┐
│                  Nginx 反向代理层                        │
│            静态资源 + API代理 + SSL终止 + 限流              │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                  NestJS 应用层                           │
├─────────────┬─────────────┬─────────────┬─────────────┤
│   认证模块   │  门店设备   │  素材节目   │  发布分发   │
│   RBAC      │   管理      │   管理      │   管理      │
│             │ 行业分类     │  审核流程    │  批量推送    │
├─────────────┼─────────────┼─────────────┼─────────────┤
│  系统管理   │  操作日志   │  多语言     │ Socket.io   │
│  (菜单/角色)│  全链路审计  │   i18n      │  推送服务   │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                    数据存储层                             │
├─────────────────────┬───────────────────────────────────┤
│      MySQL 8.0      │           Redis 7                 │
│   业务数据持久化     │    缓存 + Session + 限流         │
└─────────────────────┴───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                    文件存储层                             │
├─────────────────────────────────────────────────────────┤
│                    本地文件系统                           │
│              /data/adspread/materials/              │
│                按日期分目录存储                       │
└─────────────────────────────────────────────────────────┘
```

### 2.2 架构设计原则

1. **模块化优先**：NestJS每个功能模块独立，接口清晰，便于未来拆分微服务
2. **类型安全**：全栈TypeScript，前后端类型定义共享
3. **高内聚低耦合**：业务逻辑在Service层，Controller仅做HTTP适配
4. **可观测性**：全链路日志、请求ID追踪、性能监控
5. **渐进式扩展**：单体起步，预留未来拆分微服务的架构边界
6. **设计一致性**：统一设计系统，确保UI/UX一致性

### 4.2 核心表详细设计

#### 4.2.1 门店表 (stores)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 主键ID |
| name | VARCHAR(100) | NOT NULL | 门店名称 |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 门店编码，唯一标识 |
| industry_category | ENUM('catering', 'retail', 'beauty', 'hospitality', 'education', 'automotive', 'localLife', 'other') | NOT NULL | 行业分类：餐饮/零售/美妆/酒旅/教育/汽车/本地生活/其他 |
| address | VARCHAR(255) | NULL | 门店地址 |
| contact_person | VARCHAR(50) | NULL | 联系人 |
| contact_phone | VARCHAR(20) | NULL | 联系电话 |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：0禁用 1启用 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：** `idx_code` (UNIQUE), `idx_status`, `idx_industry_category`

**业务规则：**
- 门店编码必须全局唯一
- 行业分类为必填字段，创建时必须选择
- 门店下有设备时不允许删除
- 禁用门店下的所有设备无法接收新广告

#### 4.2.2 设备表 (devices)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 主键ID |
| store_id | INT | NULL, FOREIGN KEY | 所属门店ID，可为空 |
| name | VARCHAR(100) | NOT NULL | 设备名称 |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 设备编码，客户端标识 |
| screen_orientation | ENUM('landscape', 'portrait') | NOT NULL | 屏幕方向：landscape横屏 portrait竖屏 |
| screen_resolution | VARCHAR(20) | NOT NULL | 分辨率：1920x1080等，支持自定义 |
| split_type | VARCHAR(10) | NOT NULL | 分屏类型：1/2/3/3-1/4 |
| remark | TEXT | NULL | 备注信息（故障记录等） |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：0禁用 1启用 |
| last_active_at | DATETIME | NULL | 最后活跃时间 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：** `idx_code` (UNIQUE), `idx_store_id`, `idx_status`, `idx_last_active`

**分屏类型说明：**

| 屏幕方向 | 分屏类型 | 说明 |
|----------|----------|------|
| 竖屏 | 1 | 全屏一个区域 |
| 竖屏 | 2 | 上下两个相等区域 |
| 竖屏 | 3 | 上中下三个相等区域 |
| 横屏 | 1 | 全屏一个区域 |
| 横屏 | 2 | 左右两个相等区域 |
| 横屏 | 3 | 左中右三个相等区域 |
| 横屏 | 3-1 | 左侧占1/2，右侧上下两个各占1/4 |
| 横屏 | 4 | 田字格四个相等区域 |

#### 4.2.3 素材表 (materials)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 主键ID |
| name | VARCHAR(100) | NOT NULL | 素材名称 |
| type | ENUM('image', 'video') | NOT NULL | 类型：image图片 video视频 |
| file_url | VARCHAR(255) | NOT NULL | 文件访问URL |
| file_size | BIGINT | NOT NULL | 文件大小（字节） |
| file_extension | VARCHAR(10) | NOT NULL | 文件扩展名 |
| width | INT | NULL | 图片宽度（像素） |
| height | INT | NULL | 图片高度（像素） |
| duration | INT | NULL | 视频时长（秒） |
| audit_status | TINYINT | NOT NULL, DEFAULT 0 | 审核状态：0待审核 1审核通过 2审核驳回 |
| audit_user_id | INT | NULL, FOREIGN KEY | 审核人ID |
| audit_time | DATETIME | NULL | 审核时间 |
| audit_reason | VARCHAR(500) | NULL | 驳回原因（驳回时必填，最小10字符） |
| created_by | INT | NOT NULL, FOREIGN KEY | 上传人ID |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：** `idx_type`, `idx_audit_status`, `idx_created_by`, `idx_created_at`

**审核状态说明：**

| 状态值 | 状态名称 | 颜色标识 | 说明 |
|--------|----------|----------|------|
| 0 | 待审核 | 橙色 #fa8c16 | 素材刚上传，等待审核 |
| 1 | 审核通过 | 绿色 #52c41a | 审核通过，可在节目中使用 |
| 2 | 审核驳回 | 红色 #f5222d | 审核不通过，不可使用 |

**业务规则：**
- 素材上传后默认状态为"待审核"
- 只有"审核通过"的素材才能在节目中被选择
- 审核驳回时必须填写驳回原因（至少10个字符）
- 已审核通过的素材可以重新审核
- 支持的图片格式：JPG、PNG、GIF
- 支持的视频格式：MP4、AVI、MOV
- 单文件最大限制：100MB

#### 4.2.4 节目表 (programs)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 主键ID |
| name | VARCHAR(100) | NOT NULL | 节目名称 |
| screen_orientation | ENUM('landscape', 'portrait', 'any') | NOT NULL | 适配屏幕方向：landscape/portrait/any |
| split_type | VARCHAR(10) | NOT NULL | 适配分屏类型：1/2/3/3-1/4/any |
| layout_config | JSON | NOT NULL | 分屏区域配置 |
| status | TINYINT | NOT NULL, DEFAULT 0 | 状态：0草稿 1已发布 |
| created_by | INT | NOT NULL, FOREIGN KEY | 创建人ID |
| published_at | DATETIME | NULL | 发布时间 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**layout_config JSON结构：**
```json
{
  "regions": [
    {
      "regionId": "region1",
      "x": 0, "y": 0, "width": 0.5, "height": 1,
      "materials": [
        { "materialId": 1, "duration": 10 },
        { "materialId": 2, "duration": 15 }
      ]
    }
  ]
}
```

**业务规则：**
- 节目中只能引用审核通过的素材
- 已关联到发布计划的节目不能删除
- 草稿状态的节目可以编辑，已发布的节目需重新发布后生效

#### 4.2.5 发布计划表 (publish_plans)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 主键ID |
| program_id | INT | NOT NULL, FOREIGN KEY | 关联节目ID（只能关联已发布的节目） |
| name | VARCHAR(100) | NOT NULL | 发布计划名称 |
| target_store_ids | JSON | NOT NULL | 目标门店ID数组 [1,2,3] |
| start_time | DATETIME | NOT NULL | 生效开始时间 |
| end_time | DATETIME | NULL | 生效结束时间，null表示永久 |
| play_days | VARCHAR(20) | NOT NULL | 播放周期：1,2,3,4,5,6,7 (周一到周日) |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：0停用 1启用 |
| last_pushed_at | DATETIME | NULL | 最后推送时间 |
| created_by | INT | NOT NULL, FOREIGN KEY | 创建人ID |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：** `idx_program_id`, `idx_status`, `idx_created_by`, `idx_start_time`

**业务规则：**
- 只能选择已发布的节目创建发布计划
- 按门店发布，发布后该门店下所有设备都会接收到广告
- 播放周期为多选，广告只在选中的星期几播放
- 支持手动立即推送，推送后设备立即刷新节目

**索引：** `idx_role_menu` (UNIQUE role_id + menu_id + permission_type), `idx_role_id`, `idx_menu_id`

**设计说明：** 使用标准关系型设计，支持：
- 方便查询"某个菜单被哪些角色授权"
- 支持按钮级细粒度权限扩展
- 数据库索引优化查询效率

#### 4.2.9 管理员表 (admins)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 主键ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希（bcrypt加密） |
| name | VARCHAR(50) | NOT NULL | 姓名 |
| role_id | INT | NOT NULL, FOREIGN KEY | 关联角色ID |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：0禁用 1启用 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：** `idx_username` (UNIQUE), `idx_role_id`, `idx_status`

**业务规则：**
- 每个管理员必须关联一个角色

## 6. 实时推送设计

### 6.1 连接认证流程

```
安卓设备                      服务端
   |                            |
   |  1. WebSocket连接
   |  携带 deviceCode 和 timestamp
   |--------------------------->|
   |                            |
   |  2. 验证设备编码合法性
   |  校验时间戳防重放
   |                            |
   |  3. 连接成功
   |<---------------------------|
   |                            |
   |  4. 定期ping心跳 (30s)
   |<-------------------------->|
   |                            |
```

**连接建立参数：**
```
wss://api.adspread.com/socket.io?deviceCode=DEVICE001&timestamp=1716547200000
```

### 6.2 推送事件定义

| 事件名 | 方向 | 说明 |
|--------|------|------|
| `program_update` | 服务端→设备 | 节目更新通知，设备收到后立即拉取最新节目 |
| `push_ack` | 设备→服务端 | 设备收到推送后的确认回执 |
| `device_status` | 设备→服务端 | 设备状态实时上报 |
| `ping` / `pong` | 双向 | 心跳保活 |

**推送消息示例：**
```json
{
  "event": "program_update",
  "data": {
    "publishPlanId": 10,
    "programId": 5,
    "reason": "manual_push",
    "timestamp": 1716547200000
  }
}
```

### 6.3 推送实现机制

1. **单设备推送**：按 `deviceCode` 找到对应Socket连接，发送消息
2. **门店批量推送**：查出门店下所有设备编码，批量发送
3. **推送失败重试**：设备不在线时记录待推送队列，设备上线后补发
4. **推送日志**：所有推送操作记录到 push_message_logs 表

### 6.4 推送状态追踪

- **待推送**：已创建推送任务，设备不在线等待上线
- **已推送**：已发送到设备，等待确认回执
- **已送达**：收到设备确认回执，推送成功
- **失败**：推送失败，超过最大重试次数后标记失败

```typescript
// src/composables/useTable.ts
import { ref, reactive } from 'vue';

export function useTable<T = any>(fetchApi: Function) {
  const loading = ref(false);
  const data = ref<T[]>([]);
  const total = ref(0);
  
  const queryParams = reactive({
    page: 1,
    pageSize: 20
  });

  const fetchData = async () => {
    loading.value = true;
    try {
      const res = await fetchApi(queryParams);
      data.value = res.list;
      total.value = res.total;
    } finally {
      loading.value = false;
    }
  };

  const handlePageChange = (page: number) => {
    queryParams.page = page;
    fetchData();
  };

  const handleSizeChange = (size: number) => {
    queryParams.pageSize = size;
    queryParams.page = 1;
    fetchData();
  };

  const refresh = () => {
    queryParams.page = 1;
    fetchData();
  };

  return {
    loading,
    data,
    total,
    queryParams,
    fetchData,
    handlePageChange,
    handleSizeChange,
    refresh
  };
}
```

**分屏布局区域坐标计算：**

| 分屏类型 | 屏幕方向 | 区域坐标 (x, y, width, height) |
|----------|----------|-------------------------------|
| 1分屏 | 任意 | region1: (0, 0, 1, 1) |
| 2分屏 | 竖屏 | region1: (0, 0, 1, 0.5), region2: (0, 0.5, 1, 0.5) |

| 2分屏 | 横屏 | region1: (0, 0, 0.5, 1), region2: (0.5, 0, 0.5, 1) |

| 3-1分屏 | 横屏 | region1: (0, 0, 0.5, 1), region2: (0.5, 0, 0.5, 0.5), region3: (0.5, 0.5, 0.5, 0.5) |
| 4分屏 | 横屏 | region1: (0, 0, 0.5, 0.5), region2: (0.5, 0, 0.5, 0.5), region3: (0, 0.5, 0.5, 0.5), region4: (0.5, 0.5, 0.5, 0.5) |

```kotlin
// 分屏配置 -> 实际像素坐标转换
fun calculateRegionBounds(
  splitType: String,
  screenWidth: Int,
  screenHeight: Int
): List<RegionBounds> {
  return when (splitType) {
    "1" -> listOf(
      RegionBounds("region1", 0, 0, screenWidth, screenHeight)
    )
    "2" -> if (screenOrientation == PORTRAIT) {
      listOf(
        RegionBounds("region1", 0, 0, screenWidth, screenHeight / 2),
        RegionBounds("region2", 0, screenHeight / 2, screenWidth, screenHeight / 2)
      )
    } else {
      listOf(
        RegionBounds("region1", 0, 0, screenWidth / 2, screenHeight),
        RegionBounds("region2", screenWidth / 2, 0, screenWidth / 2, screenHeight)
      )
    }
    "3-1" -> listOf(
      RegionBounds("region1", 0, 0, screenWidth / 2, screenHeight),
      RegionBounds("region2", screenWidth / 2, 0, screenWidth / 2, screenHeight / 2),
      RegionBounds("region3", screenWidth / 2, screenHeight / 2, screenWidth / 2, screenHeight / 2)
    )
    // ... 其他分屏类型
    else -> emptyList()
  }
}
```

- 语言key命名规范：`模块名.功能名.具体文案`
- 不允许在代码中硬编码中文字符串
- 新增文案需要同时添加三种语言版本
- 数字、日期、货币格式按语言本地化处理

### 12.5 设计规范

- 遵循设计系统的CSS变量，不允许硬编码颜色值
- 间距使用标准化间距变量（--spacing-xs/sm/md/lg/xl）
- 按钮、表单、弹窗等组件统一使用Element Plus组件
- 业务页面布局保持一致：筛选栏 + 表格 + 分页
