# ADSpread Android 客户端

> 信发系统安卓门店展示端，支持多屏分屏布局、广告自动拉取、离线播放。

---

| 组件 | 技术选型 | 版本 |
|------|---------|------|
| **语言** | Kotlin | 1.9+ |
| **UI 框架** | Jetpack Compose | 1.6+ |
| **架构** | MVVM + Clean Architecture | - |
| **依赖注入** | Hilt | 2.48+ |

## 核心功能

### 1. 多屏分屏布局

| 分屏模式 | 说明 |
|---------|------|
| **全屏模式** | 100% 单视频/图片 |
| **左右分屏** | 50% : 50% 左右布局 |
| **上下分屏** | 50% : 50% 上下布局 |
| **左一右二分屏** | 33% : 67% 左右布局 |
| **三分屏** | 33% : 33% : 33% 等分布局 |
| **四分屏** | 2x2 网格布局 |

### 2. 广告播放能力

- ✅ 视频硬解码（H.264/H.265）
- ✅ 图片轮播（支持淡入淡出动画）
- ✅ 文字跑马灯
- ✅ 混合内容播放（视频+图片+文字）
- ✅ 定时切换播放列表
- ✅ 播放策略可配置（循环、定时、顺序）

### 3. 离线能力

- ✅ 广告素材预下载到本地
- ✅ 断网后继续本地播放
- ✅ 网络恢复后自动同步最新节目单
- ✅ 断点续传下载大文件

### 4. 设备管理

- ✅ 心跳上报（设备状态、播放进度、网络状态）
- ✅ 远程截图（运营后台可查看当前播放画面）
- ✅ 远程控制（重启、停止播放、音量调节）
- ✅ 定时开关机

### 5. Hermes Agent 集成

- ✅ 设备端集成轻量 Hermes Agent SDK
- ✅ 异常自动诊断和上报
- ✅ 播放策略自我优化（根据网络/设备性能）
- ✅ 设备日志智能分析和问题预警

## 与管理后台的通信协议

### 1. HTTP API 轮询

- 频率：每 5 分钟拉取一次最新节目单
- 内容：节目配置、素材列表、播放策略
- 缓存：ETag 机制，无变更时返回 304

### 2. WebSocket 实时推送

- 连接：启动后自动建立长连接
- 心跳：每 30 秒发送 Ping
- 消息类型：
  - `PROGRAM_UPDATE`: 节目单变更
  - `COMMAND_RESTART`: 远程重启设备
  - `COMMAND_SCREENSHOT`: 远程截图
  - `COMMAND_VOLUME`: 音量调节

### 3. 状态上报

- 播放进度（每 10 秒）
- 设备状态（CPU、内存、磁盘、网络）
- 异常日志（崩溃、播放失败、下载失败）

## 性能指标要求

| 指标 | 要求 |
|------|------|
| **启动速度** | 冷启动 < 3 秒 |
| **切换延迟** | 广告切换 < 200ms |
| **内存占用** | 后台播放 < 200MB |
| **CPU 占用** | 视频播放 < 15% |
| **存储空间** | 至少预留 10GB 缓存 |
| **网络容错** | 断网可连续播放 7 天 |

## Hermes Agent 能力集成

安卓端内置轻量版 Hermes Agent SDK，提供：

1. **播放策略自我优化**：根据网络状况、设备性能动态调整缓冲策略
2. **异常自动诊断**：播放失败自动分析原因，生成修复建议
3. **日志智能聚合**：异常日志自动分类和归因，上报到管理后台
4. **A/B 测试执行**：接收后台下发的播放策略实验，自动执行并上报数据
